Databinding

Databinding verbindt data uit TypeScript met de HTML-template.

Uitleg

  • Met databinding kan Angular waarden uit je component tonen of gebruiken in de HTML.
  • Je gebruikt het ook om te reageren op acties van de gebruiker, zoals klikken of typen.
  • In onze les valt property binding en style binding mee onder attribute binding.
  • Een template reference variable geeft een HTML-element een lokale naam in de template.

Belangrijke syntax

{{ '{{ pageTitle }}' }} (click)="increaseClickCount()" [(ngModel)]="studentName" [disabled]="isResetDisabled" #quickName

Examen

  • Ken het verschil tussen data tonen en reageren op events.
  • Vergeet niet dat [(ngModel)] FormsModule nodig heeft.
  • Template reference variables zijn handig voor kleine lokale acties.

Klein voorbeeld

export class Databinding {{ '{' }}
  studentName = 'Lina';
  clickCount = 0;
  isResetDisabled = true;
  templateReferenceMessage = 'Nog geen waarde opgehaald.';

  increaseClickCount() {{ '{' }}
    this.clickCount++;
    this.isResetDisabled = false;
  {{ '}' }}

  resetClickCount() {{ '{' }}
    this.clickCount = 0;
    this.isResetDisabled = true;
  {{ '}' }}

  showTemplateReferenceValue(value: string) {{ '{' }}
    this.templateReferenceMessage = value;
  {{ '}' }}
{{ '}' }}

<p>Welkom, {{ '{{ studentName }}' }}</p>
<p>Aantal clicks: {{ '{{ clickCount }}' }}</p>
<input [(ngModel)]="studentName" />
<input #quickName />
<button (click)="increaseClickCount()">Klik</button>
<button [disabled]="isResetDisabled" (click)="resetClickCount()">Reset</button>
<button (click)="showTemplateReferenceValue(quickName.value)">Lees</button>

Singleton, service laag en JSON

Een service zet data en logica apart van componenten.

Uitleg

  • Een component moet vooral data tonen en gebruikersacties opvangen.
  • Een service wordt gebruikt om data op te halen, te bewaren of te verwerken.
  • providedIn: 'root' maakt normaal 1 gedeelde instantie van de service.
  • ngOnInit is een lifecycle hook die uitgevoerd wordt wanneer het component start.

Belangrijke syntax

@Injectable({{ '{' }} providedIn: 'root' {{ '}' }}) ngOnInit() this.http.get<Student[]>(this.apiUrl) .pipe(tap(...), map(...)) catchError((error) => of([]))

Examen

  • Service laag betekent: component vraagt data aan de service, niet rechtstreeks aan JSON.
  • tap() verandert data niet, map() verandert data wel.
  • filter() haalt alleen items uit een array die voldoen aan een voorwaarde.
  • async subscribet automatisch op een Observable.
  • catchError voorkomt dat je app crasht bij API-fouten.

Klein voorbeeld

import {{ '{' }} HttpClient {{ '}' }} from '@angular/common/http';
import {{ '{' }} Injectable {{ '}' }} from '@angular/core';
import {{ '{' }} Observable, catchError, map, of, tap {{ '}' }} from 'rxjs';
import {{ '{' }} Student {{ '}' }} from '../models/student';

@Injectable({{ '{' }}
  providedIn: 'root'
{{ '}' }})
export class StudentService {{ '{' }}
  private apiUrl = 'students.json';

  constructor(private http: HttpClient) {{ '{}' }}

  getStudents(): Observable<Student[]> {{ '{' }}
    return this.http.get<Student[]>(this.apiUrl).pipe(
      tap((students) => console.log(students)),
      catchError((error) => of([]))
    );
  {{ '}' }}

  filterStudents(searchText: string): Observable<Student[]> {{ '{' }}
    return this.getStudents().pipe(
      map((students) =>
        students.filter((student) =>
          student.voornaam.toLowerCase().includes(searchText.toLowerCase())
        )
      )
    );
  {{ '}' }}
{{ '}' }}

Echte API gebruiken

Een echte API gebruikt dezelfde HttpClient-basis, maar meestal met meer endpoints.

Uitleg

  • Voor Angular lijkt een lokaal JSON-bestand ophalen sterk op data ophalen uit een echte API.
  • Een echte API staat op een server en kan vaak data toevoegen, aanpassen of verwijderen.
  • Bij een echte API kunnen beveiliging en fouten belangrijker worden.

Belangrijke syntax

HttpHeaders http.get http.post http.put http.delete

Examen

  • GET haalt data op.
  • POST maakt nieuwe data aan.
  • PUT past een volledig object aan.
  • PATCH past een deel van een object aan.
  • DELETE verwijdert data.
  • Headers, tokens, CORS en error handling komen vaak voor bij echte API's.

Voorbeeld service met echte studenten-API

import {{ '{' }} HttpClient, HttpHeaders {{ '}' }} from '@angular/common/http';
import {{ '{' }} Injectable {{ '}' }} from '@angular/core';
import {{ '{' }} Observable {{ '}' }} from 'rxjs';
import {{ '{' }} environment {{ '}' }} from '../../../environments/environment';
import {{ '{' }} Student {{ '}' }} from '../models/student';

@Injectable({{ '{' }}
  providedIn: 'root'
{{ '}' }})
export class StudentService {{ '{' }}
  apiUrl = environment.apiUrl + '/api/Students';

  httpOptions = {{ '{' }}
    headers: new HttpHeaders({{ '{' }}
      'Content-Type': 'application/json'
    {{ '}' }})
  {{ '}' }};

  constructor(private http: HttpClient) {{ '{}' }}

  getStudents(): Observable<Student[]> {{ '{' }}
    return this.http.get<Student[]>(this.apiUrl);
  {{ '}' }}

  getStudent(id: number): Observable<Student> {{ '{' }}
    return this.http.get<Student>(`${{ '${' }}this.apiUrl{{ '}' }}/${{ '${' }}id{{ '}' }}`);
  {{ '}' }}

  createStudent(student: Student): Observable<Student> {{ '{' }}
    return this.http.post<Student>(this.apiUrl, student, this.httpOptions);
  {{ '}' }}

  updateStudent(student: Student): Observable<void> {{ '{' }}
    return this.http.put<void>(`${{ '${' }}this.apiUrl{{ '}' }}/${{ '${' }}student.id{{ '}' }}`, student, this.httpOptions);
  {{ '}' }}

  deleteStudent(id: number): Observable<void> {{ '{' }}
    return this.http.delete<void>(`${{ '${' }}this.apiUrl{{ '}' }}/${{ '${' }}id{{ '}' }}`, this.httpOptions);
  {{ '}' }}
{{ '}' }}

Input & Output

Componenten kunnen data doorgeven en events terugsturen.

Uitleg

  • Parent to child betekent dat de parent data doorgeeft aan de child.
  • Child to parent betekent dat de child een event terugstuurt naar de parent.
  • Dit is handig wanneer je een groter scherm opsplitst in kleinere componenten.

Belangrijke syntax

@Input() @Output() EventEmitter $event

Klein voorbeeld

// Parent HTML
<app-student-card
  [student]="student"
  (studentSelected)="selectStudent($event)"
/>

// Parent TypeScript
export class InputOutput {{ '{' }}
  selectedStudent?: Student;

  selectStudent(student: Student) {{ '{' }}
    this.selectedStudent = student;
  {{ '}' }}
{{ '}' }}

// Child TypeScript
import {{ '{' }} Component, EventEmitter, Input, Output {{ '}' }} from '@angular/core';

@Component({{ '{' }}
  selector: 'app-student-card',
  templateUrl: './student-card.html'
{{ '}' }})
export class StudentCard {{ '{' }}
@Input() student!: Student;
@Output() studentSelected = new EventEmitter<Student>();

sendStudentToParent() {{ '{' }}
  this.studentSelected.emit(this.student);
{{ '}' }}
{{ '}' }}

Routing

Routing koppelt URLs aan componenten.

Uitleg

  • Routing zorgt ervoor dat je meerdere paginas kan tonen binnen dezelfde Angular app.
  • De browser hoeft niet volledig te herladen wanneer je naar een andere route gaat.
  • Met route parameters kan je een waarde zoals een id uit de URL lezen.

Belangrijke syntax

routerLink router-outlet students/:id ActivatedRoute

Klein voorbeeld

import {{ '{' }} Routes {{ '}' }} from '@angular/router';
import {{ '{' }} Home {{ '}' }} from './features/Home/home';
import {{ '{' }} StudentFeature {{ '}' }} from './features/Student/student';
import {{ '{' }} StudentDetail {{ '}' }} from './features/Student/student-detail/student-detail';

export const routes: Routes = [
  {{ '{' }} path: '', component: Home {{ '}' }},
  {{ '{' }} path: 'students', component: StudentFeature {{ '}' }},
  {{ '{' }} path: 'students/:id', component: StudentDetail {{ '}' }},
  {{ '{' }} path: '**', redirectTo: '' {{ '}' }}
];

<nav>
  <a routerLink="/students">Studenten</a>
</nav>
<router-outlet></router-outlet>

Reactive Form

Reactive forms worden vooral opgebouwd in TypeScript.

Uitleg

  • Bij reactive forms staat de formulierstructuur in de component class.
  • De HTML koppelt zich aan die structuur met formGroup en formControlName.
  • Validatie wordt overzichtelijk in TypeScript geplaatst met Validators.

Belangrijke syntax

FormGroup FormControl FormArray Validators.required formControlName formArrayName

Klein voorbeeld

import {{ '{' }} FormArray, FormControl, FormGroup, Validators {{ '}' }} from '@angular/forms';

export class ReactiveForm {{ '{' }}
studentForm = new FormGroup({{ '{' }}
  voornaam: new FormControl('', [Validators.required]),
  email: new FormControl('', [Validators.required, Validators.email]),
  courses: new FormArray([
    new FormControl('Front-End Development')
  ])
{{ '}' }});

get courses(): FormArray {{ '{' }}
  return this.studentForm.get('courses') as FormArray;
{{ '}' }}

saveStudent() {{ '{' }}
  if (this.studentForm.invalid) {{ '{' }}
    this.studentForm.markAllAsTouched();
    return;
  {{ '}' }}

  console.log(this.studentForm.value);
{{ '}' }}
{{ '}' }}

<form [formGroup]="studentForm" (ngSubmit)="saveStudent()">
  <input formControlName="voornaam" />
  <input formControlName="email" />

  <div formArrayName="courses">
    <input [formControlName]="0" />
  </div>

  <button [disabled]="studentForm.invalid">Opslaan</button>
</form>

Drag & Drop

Met Angular CDK kan je HTML-elementen sleepbaar maken.

Uitleg

  • cdkDrag maakt een element sleepbaar.
  • cdkDragBoundary beperkt het slepen tot een bepaald gebied.
  • cdkDragEnded laat je reageren wanneer de gebruiker stopt met slepen.

Belangrijke syntax

DragDropModule cdkDrag cdkDragBoundary cdkDragEnded

Klein voorbeeld

import {{ '{' }} CdkDragEnd, DragDropModule {{ '}' }} from '@angular/cdk/drag-drop';
import {{ '{' }} Component {{ '}' }} from '@angular/core';

@Component({{ '{' }}
  selector: 'app-drag-drop',
  imports: [DragDropModule],
  templateUrl: './drag-drop.html'
{{ '}' }})
export class DragDrop {{ '{' }}
  lastPosition = 'Nog niet verplaatst';

  onDragEnded(event: CdkDragEnd) {{ '{' }}
    const position = event.source.getFreeDragPosition();
    this.lastPosition = `X: ${{ '${' }}Math.round(position.x){{ '}' }}, Y: ${{ '${' }}Math.round(position.y){{ '}' }}`;
  {{ '}' }}
{{ '}' }}

<div class="pitch-boundary">
  <div
    class="player"
    cdkDragBoundary=".pitch-boundary"
    cdkDrag
    (cdkDragEnded)="onDragEnded($event)"
  >
    ST
  </div>
</div>
